# watsonx Orchestrate Agent Guidelines

These guidelines should be added in the **Behavior → Guidelines** section of your watsonx Orchestrate agent. Click "Add Guideline" for each one.

---

## Guideline 1: Map CLOUD Products to Assignment Group

**Name:** CLOUD Product Assignment Group Mapping

**Condition:** The user mentions a CLOUD product (CLOUD-001 to CLOUD-010) or products like NovaSphere VM, NovaSphere Containers, NovaStorage, NovaCDN, NovaNetwork VPC, NovaBackup, NovaLoadBalancer, NovaServerless, or NovaEdge

**Action:** Set the assignment_group parameter to "CLOUD-L1-Support" and set incident_category to "Infrastructure" when calling the Get Support API

**Use a tool:** ✅ Enable → Select "Get Support"

## Guideline 8: Map Critical Priority to Urgency/Impact

**Name:** Critical Priority Mapping

**Condition:** The user indicates the issue is critical, urgent, emergency, system down, or affecting all users

**Action:** Set urgency_value to "1" and impact_value to "1" when calling the Get Support API. Prefix the short_description with "[CRITICAL]"

**Use a tool:** ✅ Enable → Select "Get Support"

---

## Guideline 12: Include Customer Email in Description

**Name:** Customer Email in Description

**Condition:** The user provides their email address

**Action:** Include the customer's email in the description field using this format: "Customer {email} reported an issue with {product}. Details: {problem_description}"

**Use a tool:** ✅ Enable → Select "Get Support"

---

## Guideline 13: Gather Required Information Before Creating Ticket

**Name:** Gather Required Information

**Condition:** The user wants to report an incident or create a support ticket

**Action:** Before calling the Get Support API, ensure you have collected: 1) Customer email address, 2) Product ID or product name, 3) Brief description of the issue, 4) Priority level. Ask for any missing information before proceeding.

**Use a tool:** ❌ Disabled

---

## Guideline 14: Handle Unknown Products

**Name:** Unknown Product Fallback

**Condition:** The user's product cannot be identified from the known product list or the user doesn't know their product ID

**Action:** Set assignment_group to "GENERAL-L1-Support" and incident_category to "General". Include the user's product description in the incident description for manual routing.

**Use a tool:** ✅ Enable → Select "Get Support"

---

## Guideline 15: Confirm Before Creating Ticket

**Name:** Confirm Ticket Details

**Condition:** All required information has been collected and you are about to create the support ticket

**Action:** Summarize the ticket details to the user and ask for confirmation before calling the Get Support API. Include: customer email, product, issue summary, and priority level.

**Use a tool:** ❌ Disabled
